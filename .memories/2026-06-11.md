# MageBridgeCore — 2026-06-11

## [Candidate] [00:10] laminas/laminas-json abandoned warning assessment

**Decision: Keep status quo (Option 1) — ignore the Composer abandoned warning.**

- `laminas/laminas-json` is abandoned because it is merely a thin wrapper around PHP's native JSON functions.
- `laminas/laminas-json-server` (introduced in commit `55786ec`) is still **actively maintained** and fully compatible with PHP 8.3/8.4.
- Rolling back to the old embedded Zend 1 (`libraries/Zend/`) is not an option — it causes fatal errors on PHP 8.3.
- Replacing or self-implementing a JSON-RPC server carries unacceptably high regression risk (SMD format is tightly coupled to the Magento-side API).
- Conclusion: this warning is cosmetic; it has zero security or performance impact at runtime. No action required.

## [Candidate] [00:10] saloonphp/saloon security vulnerabilities (Dependabot #28–30)

**Decision: No action required for MageBridgeCore.**

- 3 Dependabot alerts — #28 Medium (SSRF / credential leak via absolute URL), #29 Medium (fixture-name path traversal), #30 High (unsafe deserialization in `AccessTokenAuthenticator`) — all originate from `saloonphp/saloon`.
- Dependency chain: `openmage/magento-lts` (require-dev) → `shipstream/fedex-rest-sdk` → `saloonphp/saloon ^3.x`.
- This is a **dev-only indirect dependency**; it is never deployed to the production Joomla extension.
- Fix requires upgrading `saloonphp/saloon` to v4.0.0+ (breaking change); must wait for upstream `shipstream/fedex-rest-sdk` to release a compatible version.

Tracking items (not MageBridgeCore's responsibility):
- [ ] OpenMage production sites: verify whether the FedEx REST carrier is enabled. If so, ensure the cache backend (Redis/Memcached) has proper access controls — alert #30 carries real RCE risk in that scenario.
- [ ] Monitor `shipstream/fedex-rest-sdk` for a release that supports `saloonphp/saloon ^4.0`.

Source: `.agents/todo/saloon_vulner.md`

## [Candidate] [00:13] Dependabot configuration — grouped updates + supply-chain cooldown

Updated `.github/dependabot.yml` to reduce PR noise and harden against supply-chain attacks:

- Added `groups` with `patterns: ["*"]` to all three ecosystems (`composer`, `github-actions`, `npm`) so each ecosystem produces at most one PR per cycle.
- Added `cooldown: default-days: 3` to all ecosystems so Dependabot waits 3 days after a package is published before opening a PR, giving the community time to detect and report malicious releases.
